## Debug Session: Vercel Build Failure — `PrismaClient` not exported

**Session ID:** 20260320_155242_136203  
**Error:** `Type error: Module '"@prisma/client"' has no exported member 'PrismaClient'`

---

## Root Cause

Prisma v7 generates its TypeScript types into `.prisma/client/default.d.ts` at generate-time (not bundled inside `node_modules/@prisma/client`). `@prisma/client/index.d.ts` re-exports from that generated file:

```ts
export * from '.prisma/client/default';
```

Because `prisma generate` was never called on Vercel (no `postinstall` or build-time step), the generated type file did not exist. TypeScript therefore could not resolve `PrismaClient`, causing the build to fail. The runtime JS worked fine because `@prisma/client/index.js` bundles a fallback runtime.

---

## Changes Made

### 1. Added `prisma generate` to the build script

- **File:** `package.json`
- **Change:** Modified `"build"` from `"next build"` → `"prisma generate && next build"`
- **Why:** Ensures the Prisma client types are generated before `next build` runs the TypeScript compiler on Vercel (and locally via `pnpm run build`).
- **Revert:** Change `"build"` back to `"next build"`
